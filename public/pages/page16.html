Polimorfizm

Polimorfizm, obyektga yo'naltirilgan dasturlashda, bir usul yoki ob'ektning bir necha turli xil shakllarda paydo bo'lishini anglatadi. Java'da bu, bir sinfning bir nechta o'zgarishlarini amalga oshirish yoki bir interfeysning bir nechta sinflar tomonidan amalga oshirilishi orqali amalga oshiriladi.

Java'da polimorfizmni qanday amalga oshirish haqida oddiy bir misol keltiraman: 


        public class Hayvon {
            public void ovozBer() {
                System.out.println("Hayvon tovush chiqaradi.");
            }
        }

        public class It extends Hayvon {
            @Override
            public void ovozBer() {
                System.out.println("It vov-vov qiladi.");
            }
        }
        
        public class Mushuk extends Hayvon {
            @Override
            public void ovozBer() {
                System.out.println("Mushuk miyovlaydi.");
            }
        }
        
        public class Main {
            public static void main(String[] args) {
                Hayvon it = new It();
                Hayvon mushuk = new Mushuk();
                
                it.ovozBer();   // It vov-vov qiladi
                mushuk.ovozBer(); // Mushuk miyovlaydi
            }
        }

Ushbu misolda, Hayvon sinfi umumiy xususiyat va metodlarni o'z ichiga oladi. It va Mushuk sinflari esa, Hayvon sinfidan meros olib, ovozBer metodini qayta belgilaydi (override qiladi) va o'ziga xos xatti-harakatni ifodalaydi. Polimorfizm orqali, bir usul turli xil obyektlar bilan turli xil xatti-harakatlarni amalga oshirishi mumkin.
