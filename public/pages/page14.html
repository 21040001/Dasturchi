Inkapsulyatsiya

Inkapsulyatsiya, Java va boshqa obyektga yo'naltirilgan dasturlash tillarida, ma'lumotlar va ularni qayta ishlovchi usullarni birgalikda paketlashni anglatadi. Bu konsept, ma'lumotlarga tashqi tomondan to'g'ridan-to'g'ri kirishni oldini olish orqali faqatgina belgilangan usullar orqali kirishni ta'minlaydi. Shu tariqa, ma'lumotlarning butunligi va xavfsizligi saqlanadi.

Java'da inkapsulyatsiyani qanday amalga oshirish haqida oddiy bir misol keltiraman: 


        public class Student {
            // Xususiy (private) maydonlar
            private String ism;
            private int yosh;
        
            // Ommaviy (public) kirish usullari (getter va setter)
            public String getIsm() {
                return ism;
            }
        
            public void setIsm(String yangiIsm) {
                this.ism = yangiIsm;
            }
        
            public int getYosh() {
                return yosh;
            }
        
            public void setYosh(int yangiYosh) {
                if (yangiYosh > 0) {
                    this.yosh = yangiYosh;
                }
            }
        }
        
        public class Main {
            public static void main(String[] args) {
                Student student = new Student();
                student.setIsm("Ahmad");
                student.setYosh(20);
        
                System.out.println("Talabaning ismi: " + student.getIsm());
                System.out.println("Talabaning yoshi: " + student.getYosh());
            }
        }

Bu misolda, ism va yosh o'zgaruvchilari private deb belgilangan, ya'ni bu maydonlarga to'g'ridan-to'g'ri kirish cheklangan. Biroq, getIsm, setIsm, getYosh va setYosh kabi public kirish usullari (getter va setter) orqali bu maydonlarga kirish mumkin.
